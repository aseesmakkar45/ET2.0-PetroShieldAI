# Contributing to PetroShield AI

First off, thank you for considering contributing to PetroShield AI.

## Branch Naming Convention

We follow a structured branch naming convention to keep our repository organized. Please prefix your branch with the type of work being done:

- `feature/` for new features (e.g., `feature/scenario-engine-updates`)
- `bugfix/` for bug fixes (e.g., `bugfix/map-rendering-issue`)
- `docs/` for documentation changes (e.g., `docs/update-readme`)
- `chore/` for maintenance tasks (e.g., `chore/update-dependencies`)

## Commit Message Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification.

Format: `<type>(<scope>): <subject>`

**Examples**:
- `feat(map): add vessel tracking overlays`
- `fix(auth): resolve JWT expiration logic`
- `docs(readme): add installation instructions`

**Types**:
- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)
- `refactor`: A code change that neither fixes a bug nor adds a feature
- `perf`: A code change that improves performance
- `test`: Adding missing tests or correcting existing tests
- `chore`: Changes to the build process or auxiliary tools and libraries

## Pull Request Workflow

1. Fork the repository and create your branch from `main`.
2. Ensure your code follows the established coding standards.
3. Test your changes thoroughly.
4. Issue a Pull Request filling out the provided PR template.
5. Link any related issues in the PR description.
6. Request review from maintainers.
7. Address any feedback and push updates to your branch.
